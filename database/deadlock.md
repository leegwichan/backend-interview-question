# 데드락 (Deadlock)
- 두 개 이상의 프로세스가 서로 점유하고 있는 자원의 해제를 무한히 기다리는 교착 상태

### 데드락 발생의 4가지 필요 조건
- 상호 배제 : 자원은 한 번에 하나의 프로세스만 사용할 수 있어야 한다.
- 점유와 대기 : 자원을 점유하고 있는 프로세스가 다른 자원을 추가로 얻기 위해 대기해야 한다.
- 비선점 : 프로세스가 점유한 자원을 강제로 빼앗을 수 없어야 하며, 오직 점유한 프로세스 스스로가 해제해야 한다.
- 순환 대기 : 대기 중인 프로세스들이 자원을 얻기 위해 서로 물고 물리는 형태로 순환하며 기다려야 한다.

### 데드락 해결 및 방지 방법
- 예방 : 위 4가지 조건 중 하나를 미리 위반하도록 설계하여 데드락 발생 자체를 막는 것
  - ex. 순환 대기 조건을 막기 위해 자원 할당 순서를 정하는 것
- 회피 : 자원 할당 시 데드락이 발생할 가능성이 있는지 미리 확인하고, 안전한 상태에서만 자원을 할당하는 방법
  - ex. Banker's Algorithm : 자원 할당 요청이 들어올 때마다, 시스템을 불안전 상태로 만들지 않는지를 미리 검사 (최대 필요 자원, 현재 할당 자원, 추가 필요 자원, 사용 가능 자원을 모두 알고 있어야 한다)
- 탐지 및 복구 : 시스템이 데드락을 이를 탐지한 후, 희생자(Victim) 프로세스를 선정하여 강제로 종료시키고 자원을 해제시켜 데드락을 푸는 방법입니다.
  - ex. InnoDB의 잠금 대기 그래프 : 트랜잭션과 잠금 관계를 나타내는 방향성 그래프로 사이클이 형성되는 순단을 감지
